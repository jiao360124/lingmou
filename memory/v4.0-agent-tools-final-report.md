## 核心架构实现

### 1. 插件入口文件 (`core/plugin-entry.js`)

**功能**:
- 动态扫描 `core/monitoring/` 目录
- 动态扫描 `core/strategy/` 目录
- 为每个模块创建 `api.registerTool()` 包装
- 实现异步工具执行函数

**代码结构**:
```javascript
module.exports = function(api) {
  // 加载监控模块（3个）
  const monitoringPath = path.join(__dirname, 'monitoring');
  const monitoringModules = fs.readdirSync(monitoringPath)...;

  // 加载策略引擎模块（12个）
  const strategyPath = path.join(__dirname, 'strategy');
  const strategyModules = fs.readdirSync(strategyPath)...;

  // 注册每个模块为Agent Tool
  monitoringModules.forEach(moduleName => {
    const tool = createPluginWrapper(
      `monitoring_${moduleName}`,
      modulePath,
      description
    );
    api.registerTool(tool);
  });

  // 重复策略引擎模块注册
};
```

**工具命名规范**:
- 监控模块: `monitoring_<module_name>`
- 策略引擎: `strategy_<module_name>`

### 2. 插件配置 (`openclaw-plugin.json`)

```json
{
  "name": "core-modules",
  "version": "1.0.0",
  "description": "OpenClaw v4.0 核心模块插件 - 注册策略引擎和监控模块为 Agent Tools",
  "openclaw": {
    "entry": "./core/plugin-entry.js",
    "type": "agent-tool",
    "enabled": true
  }
}
```

## 工具调用流程

### Agent工具调用流程图

```
用户消息
  ↓
Agent接收消息
  ↓
加载上下文（工具Schema + Skills）
  ↓
LLM决策（基于工具Schema选择工具）
  ↓
返回工具调用请求
  ↓
OpenClaw执行工具调用
  ↓
插件入口加载模块实例
  ↓
调用模块的getStatus()方法
  ↓
返回JSON格式结果
  ↓
返回给Agent
  ↓
Agent继续生成回复
```

### 实际调用示例

**测试提示词**: "请获取当前系统性能监控状态"

**预期流程**:
1. Agent识别到 `monitoring_performance-monitor` 工具
2. Agent准备工具调用参数: `{}`
3. Agent调用工具: `monitoring_performance-monitor({})`
4. 插件加载 `performance-monitor.js`
5. 插件创建模块实例
6. 插件调用 `instance.getStatus()`
7. 返回JSON: `{"success": true, "status": "OK", ...}`
8. Agent将结果发送给LLM
9. LLM生成回复

## 核心架构要点

### ✅ 已解决的问题

1. **JS模块 ≠ Agent Tools**
   - 问题: 核心模块只是普通JS文件
   - 解决: 创建插件入口，使用`api.registerTool()`包装

2. **工具调用机制**
   - 问题: Agent不知道工具存在
   - 解决: 通过`api.registerTool()`向LLM暴露工具Schema

3. **技能配置**
   - 问题: Skills未在openclaw.json中声明
   - 解决: 添加所有44个技能到`skills.entries`

4. **动态加载**
   - 问题: 需要重启Gateway才能加载新模块
   - 解决: Gateway自动检测配置变化并重新加载

### 🔑 关键理解

**OpenClaw架构三层**:
1. **核心层**: JS模块（core/monitoring/, core/strategy/）
2. **插件层**: 插件入口，使用`api.registerTool()`包装
3. **Agent层**: Agent通过LLM的上下文访问工具

**Agent只能访问"知道存在"的东西**:
- 工具Schema必须通过`api.registerTool()`注册
- Skills必须正确配置HAML格式
- 配置必须在openclaw.json中声明
