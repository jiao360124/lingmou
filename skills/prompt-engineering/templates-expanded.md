## 文档写作模板（15+）

### 1. 技术文档

#### API文档
```markdown
# API文档

## 概述
本文档描述[API名称]的接口规范。

## 基础信息
- **Base URL**: https://api.example.com
- **版本**: v1
- **协议**: HTTPS

## 认证
使用JWT Token进行认证。

### 获取Token
```http
POST /auth/login
Content-Type: application/json

{
  "username": "user@example.com",
  "password": "password"
}

Response:
```json
{
  "access_token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "token_type": "bearer",
  "expires_in": 3600
}
```

## API端点

### 获取用户列表
```http
GET /users
Authorization: Bearer {token}
```

**查询参数**:
- page (int): 页码，默认1
- limit (int): 每页数量，默认20
- sort (string): 排序字段，默认created_at

**响应**:
```json
{
  "data": [
    {
      "id": 1,
      "name": "John Doe",
      "email": "john@example.com",
      "created_at": "2024-01-01T00:00:00Z"
    }
  ],
  "meta": {
    "total": 100,
    "page": 1,
    "limit": 20,
    "pages": 5
  }
}
```

**错误码**:
- 401: 未授权
- 400: 请求参数错误
- 404: 资源不存在

## 错误处理
所有错误响应遵循以下格式：

```json
{
  "error": {
    "code": "INVALID_PARAMETER",
    "message": "Invalid parameter value",
    "details": {
      "field": "email",
      "expected": "valid email format"
    }
  }
}
```
```
