📝 使用示例
================================================================================

1. 技能懒加载
```javascript
const lazyLoader = require('./core/lazy-loader');

const loader = new lazyLoader();
loader.preloadSkills(['code-mentor', 'database']);
console.log(`已加载技能: ${loader.getLoadedCount()}`);
```

2. 异步I/O
```javascript
const asyncIO = require('./core/async-io');

const io = new asyncIO();

// 异步读取
const content = await io.readFileAsync('file.txt');

// 异步写入
await io.writeFileAsync('file.txt', content);

// 批量操作
const results = await io.batchExecute([
  () => io.readFileAsync('file1.txt'),
  () => io.readFileAsync('file2.txt')
], 5);
```

3. API缓存
```javascript
const apiCache = require('./core/api-cache');

const cache = new apiCache('./core/api-cache-config.json');

// 设置缓存
cache.set('gateway-status', data, 300);

// 获取缓存
const data = cache.get('gateway-status');

// 查看统计
const stats = cache.getStats();
console.log(`缓存大小: ${stats.size}`);

// 清除缓存
cache.clear();
```

4. 策略引擎优化
```javascript
const optimizedStrategyEngine = require('./core/optimized-strategy-engine');

const engine = new optimizedStrategyEngine();

// 执行策略
const result = await engine.executeStrategy('my-strategy', {
  param1: 'value1'
});

// 查看统计
const stats = engine.getStats();
console.log(`缓存统计:`, stats);

// 清除缓存
engine.clearCaches();
```

================================================================================
⚠️ 注意事项
================================================================================

1. 缓存一致性
   - API缓存可能导致数据延迟
   - 建议定期清除缓存: cache.clear()
   - 监控缓存大小避免内存溢出

2. 路径配置
   - 所有模块使用相对路径
   - core/index.js 作为统一入口点
   - 建议通过 core 模块导入，避免直接导入

3. 依赖关系
   - lazy-loader 依赖 skills 目录
   - api-cache 依赖 api-cache-config.json
   - optimized-strategy-engine 依赖 strategy 目录

4. 错误处理
   - 所有模块包含错误处理
   - 缺少依赖会抛出明确错误
   - 建议在生产环境添加 try-catch
